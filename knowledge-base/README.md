# AI渣男识别器知识库总览

本目录存放 AI 渣男识别器的公开方法说明：分类口径、评分规则、文献映射、案例材料和输出格式约束。

当前版本是纯 Beta 测试阶段的玩具项目，分析结果不带有任何参考价值，不应作为关系判断、心理判断、法律判断或现实行动依据。

用户输入一段聊天记录或恋爱经历后，识别器应输出：

1. 渣男指数：0-100 分。
2. 8 大类典型话术命中情况。
3. 逐条证据：引用原文片段，而不是凭感觉下结论。
4. 心理操控机制：说明它如何影响判断、自尊、边界、依恋或承诺感。
5. 止损建议：不替用户决定，只给风险边界和下一步观察/沟通/求助建议。

## 推荐架构

```text
用户文本
  -> 预处理与隐私清洗
  -> 证据抽取
  -> 8类话术分类
  -> 心理机制解释
  -> 0-100分风险评分
  -> 止损/沟通/求助建议
```

知识来源分三层：

1. 规则层：8 类话术 taxonomy、证据强度、上下文缺口和反误判规则。
2. 学术层：本目录 `literature-map.md` 收录的心理学、社会学、亲密关系研究。
3. 产品层：`detector-taxonomy-and-scoring.md` 与 `prompt-output-contract.md` 固化为可执行 schema。

## 文件说明

- `literature-map.md`：8 类话术对应的论文/权威资料地图，含扩展来源补充。
- `deep-analysis-methods.md`：深度分析流程、风险评估、量表映射和反事实检查。
- `casebook.md`：20 个公开案例/案例型材料卡片，按本项目 8 类话术、安全覆盖和反误判规则映射。
- `case-expansion-research.md`：公开案例库扩展清单、第二批扩展记录、来源分层、入库标准和用户自愿案例边界。
- `detector-taxonomy-and-scoring.md`：分类定义、证据标准、0-100 分评分。
- `weight-rationale.md`：权重来源、三轴归一化方法和局限性。
- `prompt-output-contract.md`：可直接交给 LLM 的系统提示词与 JSON 输出契约。

## 产品边界

这个识别器不做人格诊断，不给“他一定是某种人格障碍”的结论，也不替用户做分手/复合决定。当前输出只是玩具式文本模式展示，不具备参考价值。

如果输入中出现暴力威胁、跟踪、强迫性行为、自伤威胁、隔离控制、财务控制等安全风险，评分不应成为重点，应直接触发安全建议：联系可信任的人、保存证据、寻求本地专业支持或紧急服务。
